## Root Cause Analysis

The main problem appears to be **extremely slow electronic convergence** in the Ce7O12 system. Here's why:

1. **Electronic Structure Complexity:** Ce7O12 has a complex electronic structure with strong correlation effects from the Ce 4f electrons and the LDA+U correction

2. **LDA+U Convergence Issues:** The `LDAU = .TRUE.` with `U = 5.0` for Ce creates challenging self-consistency problems

3. **Magnetic Moment Initialization:** The `MAGMOM = 21*5.0 36*0.6` initialization may be far from the ground state

4. **System Size:** 57 atoms is relatively large for AIMD, and the electronic structure is not converging efficiently

### Try to Get Single SCF Loop to Converge

Pause the AIMD and make one (or a few) static SCF calculations converge first. Once you have a clean, well‑localized 4f solution and a sensible magnetic pattern, you can carry that WAVECAR/CHGCAR into MD and most of the “10–20 minutes per ionic step” pain disappears. Below is a compact, battle‑tested way to do it on ceria‑derived cells.

Your symptoms—RMM/DAV flipping, hard NELM caps, and dramatic slowdowns that don’t improve with ALGO changes—are classic for mixed‑valence Ce with +U when the starting spin/charge configuration is far from any stable 4f solution. In your base INCAR the biggest red flags are the magnetic seeding (`MAGMOM = 21*5.0 36*0.6`) and trying to run MD before a robust SCF has formed localized Ce³⁺ 4f¹ states. For Ce₇O₁₂ you want 12 Ce³⁺ (≈1 μB) and 9 Ce⁴⁺ (≈0 μB) in the 21‑Ce small cell; O should start at 0 μB. Starting there massively reduces the solver’s flailing.


Populate `ARCH_10`, `ARCH_11`, and `ARCH_12` with **static SCF** variants as above, fix the MAGMOM seeding to 12 Ce³⁺/9 Ce⁴⁺ for the small cell, and drop `NPAR` in favor of `NCORE`. Once one SCF converges cleanly, reuse that WAVECAR/CHGCAR for AIMD with Davidson, modest smearing, and a slightly higher NELM. That sequence is the fastest path to turning your 10–20 min/step into something sane.

#### Minimal SCF plan (run 2–3 variants in ARCH_10/11/12)

Keep the **57‑atom** cell for speed and do **static** runs (`NSW=0`). Delete any leftover WAVECAR/CHGCAR in these test directories so each variant is clean.


##### Common setup (put this in each test INCAR unless overridden below)

```ini title:INCAR.base
# ----- electronic & SCF -----
ENCUT     = 520           # a bit higher than 400 improves robustness
PREC      = Accurate
EDIFF     = 1E-6
ISPIN     = 2
ISMEAR    = 0             # Gaussian; safe for insulator/small-gap
SIGMA     = 0.05
LREAL     = .FALSE.       # disable real-space projection for clean SCF
NELM      = 200
NELMDL    = -12           # no charge mixing for first 12 iters (stabilizes)
LASPH     = .TRUE.
LMAXMIX   = 6             # include f in charge mixing (crucial for Ce)
ISYM      = 0
ICHARG    = 2
NSW       = 0
LWAVE     = .TRUE.
LCHARG    = .TRUE.

# ----- DFT+U (Dudarev) -----
LDAU      = .TRUE.
LDAUTYPE  = 2
LDAUL     = 3  -1
LDAUU     = 5.0  0.0
LDAUJ     = 0.0  0.0
LDAUPRINT = 1

# ----- k-points (either specify in INCAR or provide a KPOINTS file) -----
KSPACING  = 0.45         # ~Γ-centered ~2×2×2 for your small cell
KGAMMA    = .TRUE.

# ----- parallel -----
# Prefer NCORE in VASP 6; remove NPAR.
NCORE     = 4            # on 64 MPI ranks, 4–8 are usually good starting points

# ----- MAGMOM seeding for Ce21 O36 -----
# 12 Ce3+ ~1 μB, 9 Ce4+ ~0 μB, O ~0 μB
MAGMOM    = 12*1.0 9*0.0 36*0.0
```

> If you know which Ce sites should be Ce³⁺ (e.g., near vacancies), reorder the 12×1.0 entries to match those site indices; if not, the generic seeding above is still far better than “21×5.0”.

Now run **two or three** short SCF variants side‑by‑side:


##### ARCH_10 — “Robust Davidson”

```ini
ALGO      = Normal        # blocked Davidson; slower than RMM but sticky
```

This is the workhorse and often converges cleanly once the MAGMOMs are sane.

##### ARCH_11 — “All-electron optimizer”

```ini
ALGO      = All           # hybrid Davidson/RMM workflow (robust on f-systems)
```

This can be even more forgiving on awkward starting guesses than Normal.

##### ARCH_12 — “Nuclear option for stubborn cases”

```ini
ALGO      = Damped
TIME      = 0.3           # 0.2–0.5 is typical; smaller = more damping
```

`ALGO=Damped` is slow per iteration but almost unkillable for nasty 4f+U systems. Use it only if ARCH_10/11 don’t settle.

> **Only if you still see oscillations after these:** gently _dampen_ mixing rather than making it vanishingly small. Try:
> 
> ```
> AMIX = 0.02
> BMIX = 0.8
> AMIX_MAG = 0.04
> BMIX_MAG = 1.6
> ```
> 
> Avoid extreme values like `BMIX=1e-4`; that often makes things _slower_, not faster, by under‑preconditioning long‑wavelength sloshing.



#### After you get one clean SCF

1. Inspect `OUTCAR`/`OSZICAR` to confirm the site‑projected 4f occupancy and local moments: you want ~1 μB on 12 Ce and ~0 on 9 Ce. If needed, reassign which Ce are seeded at 1.0 and rerun the SCF once to lock in a physically sensible polaron pattern.
    
2. Use that **WAVECAR/CHGCAR** to seed AIMD. Keep the settings that converged SCF best, but relax a bit for MD:
    
    - `EDIFF = 1E-4`
        
    - `ALGO = Normal` (stick with Davidson; it’s more stable for +U MD than `Fast`)
        
    - `ISMEAR = 1`, `SIGMA = 0.2` (broadened smearing helps MD step‑to‑step continuity)
        
    - `NELM = 120–160` and keep `NELMDL = -6` if you saw early‑iter churn.
        
    - Keep `LREAL = Auto` for the **large** 152‑atom cell to save time, but prefer `.FALSE.` in the small cell and for the **initial SCF**.
        
3. Leave the thermostat alone (`SMASS=1`) and keep `POTIM = 1.0 fs`. Your earlier `SMASS=0.3` test likely made the electronic noise from poor SCF more visible.
    



#### A few targeted cleanups to your base files

- **Magnetic seeding**: update `MAGMOM` in `INCAR.base` to the Ce³⁺/Ce⁴⁺ pattern; remove magnetization on O:
    
    ```

    MAGMOM = 12*1.0 9*0.0 36*0.0
    ```
    
    For the 152‑atom cell (Ce56 O96), scale accordingly: Ce³⁺ count is 4/74/7 of Ce sites → **32 Ce³⁺** and **24 Ce⁴⁺**:

- **Keep `LMAXMIX=6`** and `LASPH=.TRUE.` (both are right for Ce +U).

#### How I’d lay out the test directories

Create three siblings with just POSCAR/POTCAR/KPOINTS and an INCAR per the variants above:

```bash

#### If convergence still crawls

Two extra tricks that are often decisive on ceria:

1. **Pre‑converge without +U, then turn +U on**. Do a single SCF with `LDAU=.FALSE.`, `ISMEAR=1`, `SIGMA=0.2`, `ALGO=Normal`, `NELM=120`, write WAVECAR/CHGCAR, then restart with +U **and** the good MAGMOM seeding. This settles the itinerant part of the charge before you localize 4f.
    
2. **Use `ALGO=Damped` just to get a first +U solution**, then switch back to `Normal` and reconverge from that WAVECAR. Damped is a ladder out of bad basins; Davidson is the highway once you’re out.
    


### Other Adjustments

1. **Increase NELM:** Try `NELM = 200` or higher to allow more electronic iterations
2. **Adjust LDA+U:** Consider reducing `U` from 5.0 to 3.0-4.0 for Ce
3. **Better Initial Guess:** Use a converged electronic structure from a static calculation as starting point
4. **Mixing Parameters:** Try `AMIX = 0.1` and `BMIX = 0.0001` for better convergence
5. **Step-by-Step Approach:** Run a static SCF calculation first, then use the converged WAVECAR/CHGCAR for AIMD

This ARCH runs four independent canonical (NVT) AIMD replicas for CeO₂ at 250 K, 500 K, 750 K, and 1000 K using VASP 6 on PSC Bridges-2. ### What this directory contains

- `submit_chain.sh` — submits 10 successive 1,000-step chunks per temperature with `afterok` dependencies.
- `250/`, `500/`, `750/`, `1000/` — per-temperature working dirs created at run time; contain archived files like `OUTCAR_*`, `OSZICAR_*`, `XDATCAR_*`, `CONTCAR_<total>.vasp`, and optional `WAVECAR_*`, `CHGCAR_*`.
- `convergence.py` and `analyze_convergence.py` — parsers/plotters for MD temperature tracking and free-energy drift. They write `convergence_analysis.png` and `convergence_summary.csv`.

### What “failure” looks like

- **Thermostat runaway or mis-tuned Nose–Hoover**: `T_mean_K` far from the target, `T_std_K` exploding, or temperature trending instead of fluctuating. Check `SMASS`, `POTIM`, and that `TEBEG = TEEND = target`.
- **Large energy drift**: |ΔF| ramping steadily (e.g., ≳ 0.1 eV per 10 k steps) indicates real-space projection (`LREAL`) issues, too-coarse grids, or too-aggressive time step. See “Stability tweaks” below.
- **SCF non-convergence**: frequent `NELM` hits, wild charge sloshing, or “ZHEGV”/diagonalization crashes. Consider a more robust ALGO and gentler mixing.
- **Chain breaks**: a chunk fails and later chunks never start; you’ll see missing `CONTCAR_<step>.vasp` for the expected step count in that temperature directory.
    

### How the submission works

Each chunk uses `vasp_submit <TEMP> <CHUNK>`, where `TEMP ∈ {250, 500, 750, 1000}` and `CHUNK` is zero-based. The script:

- Creates/enters the per-temperature directory.
- If `CHUNK == 0`, copies master inputs; otherwise it loads `CONTCAR_<CHUNK*NSW>.vasp` as the restart POSCAR and, when `KEEP_WAVE=1`, warms from the previous `WAVECAR_*` and `CHGCAR_*`.
- Builds `INCAR` by appending:
    
    ```
    TEBEG = <TEMP>
    TEEND = <TEMP>
    NSW   = <NSW>
    ```
    
- Runs VASP with MPI, then archives outputs per block and writes a cumulative checkpoint `CONTCAR_<total_steps>.vasp`.

`submit_chain.sh` just loops over temps and queues 10 chunks with `--dependency=afterok:<jobid>` so each subsequent chunk starts only after the previous one succeeds.

### Common troubleshooting

- **“missing checkpoint CONTCAR_.vasp”** when starting `CHUNK>0`: the prior chunk didn’t finish or was cleaned. Re-run the previous chunk to regenerate the checkpoint, or use the most recent `CONTCAR_<nearest_lower_step>.vasp` and adjust `CHUNK` accordingly.
    
- **Jobs start but hang at SCF**: try `ALGO=Normal`, increase `NELM`, and apply gentler `AMIX/BMIX`. Verify `ENCUT` and PAW set are consistent with your `POTCAR`.
    
- **Thermostat oscillations**: test `SMASS=2` or reduce `POTIM`. Confirm `TEBEG=TEEND`.
    
- **High drift**: switch `LREAL=Auto`, add `ADDGRID=.TRUE.`, or temporarily set `PREC=Accurate` to test sensitivity. If drift falls, keep the more conservative setting.

#### 2) Sample frames from your MD and stage SP jobs

This Python script walks `250/ 500/ 750/ 1000/`, grabs every Nth step from each `XDATCAR_*`, and writes `sp/<TEMP>/frame_XXXX/POSCAR`. Adjust `STRIDE` to get ~300–600 frames per temperature (good starting set).

Create `make_sp_dataset.py` in `11_ARCH/`:

```python
#!/usr/bin/env python3
from pathlib import Path
import re

STRIDE = 20  # 10k steps → ~500 frames per temp

temps = ["250","500","750","1000"]
xdat_pat = re.compile(r"XDATCAR_(\d+)K_blk(\d+)\.xdatcar$", re.IGNORECASE)

def split_xdatcar(p, outroot, stride):
    lines = p.read_text().splitlines()
    # XDATCAR header (fixed across blocks)
    header = lines[:7]
    # Find atom count from header
    counts = list(map(int, header[6].split()))
    nat = sum(counts)
    # Each frame starts with a line like "Direct configuration=     123"
    idxs = [i for i,l in enumerate(lines) if l.strip().startswith("Direct configuration=")]
    out = []
    for k, i0 in enumerate(idxs):
        coords = lines[i0+1 : i0+1+nat]
        if len(coords) < nat: break
        step_str = lines[i0].split("=")[-1].strip()
        step = int(step_str) if step_str.isdigit() else k
        if step % stride != 0: continue
        d = outroot / f"frame_{step:05d}"
        d.mkdir(parents=True, exist_ok=True)
        (d/"POSCAR").write_text("\n".join(header + ["Direct"] + coords) + "\n")
        out.append(d)
    return out

root = Path(".")
sp_root = root / "sp"
sp_root.mkdir(exist_ok=True)

all_frames = []
for T in temps:
    tdir = root / T
    if not tdir.is_dir(): 
        print(f"skip {T}: not found"); 
        continue
    for f in sorted(tdir.glob("XDATCAR_*")):
        m = xdat_pat.match(f.name)
        if not m: 
            continue
        frames = split_xdatcar(f, sp_root/ T, STRIDE)
        all_frames.extend(frames)
        print(f"{T}: {f.name} → {len(frames)} frames")

print(f"Total frames staged: {len(all_frames)}")
```

Run:

```bash
python3 make_sp_dataset.py
```

You’ll get `sp/250/frame_00000/POSCAR`, … etc.

#### 3) Submit a scalable SP job array

This creates one SLURM array over all staged frames and runs accurate SP for each.

`vasp_sp.sh`:

```bash
#!/usr/bin/env bash
#SBATCH -p RM-shared
#SBATCH -N 1
#SBATCH --ntasks-per-node=64
#SBATCH -t 08:00:00
#SBATCH -A mat220011p
#SBATCH -J CeO2_SP
#SBATCH -o sp_%A_%a.out

set -euo pipefail
module purge
module load intel/20.4 intelmpi/20.4-intel20.4
export OMP_NUM_THREADS=1
vasp_exec=/opt/packages/VASP/VASP6/INTEL/vasp_std

# Map array index to a frame directory
mapfile -t LIST < <(find sp -type f -name POSCAR | sort)
idx=${SLURM_ARRAY_TASK_ID:?}
target=${LIST[$idx]}
framedir=$(dirname "$target")
echo "Labeling $framedir"

workdir="$framedir/run"
mkdir -p "$workdir"
cp -f "$framedir/POSCAR" "$workdir/"
cp -f INCAR.sp "$workdir/INCAR"
cp -f POTCAR KPOINTS "$workdir/"

cd "$workdir"
mpirun -np "${SLURM_NTASKS:-64}" "$vasp_exec"

# keep minimal artifacts next to POSCAR
for f in OUTCAR OSZICAR vasprun.xml; do
  [[ -f $f ]] && cp -f "$f" "$framedir/$f"
done
```

Submit:

```bash
# count frames
N=$(find sp -type f -name POSCAR | wc -l)
sbatch --array=0-$((N-1)) vasp_sp.sh
```
